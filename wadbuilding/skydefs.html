SKYDEFS Text Lump

SKYDEFS is used to define properties of a sky background for the PC engines of Doom 64. It is defined as a text file, specifically "SKYDEFS.txt". Each sky definition must have a corresponding texture of the same name to be usable in-game. The contents of the texture is not important.

Sky Definitions

A sky definition begins with the keyword "sky", followed by the sky texture name, and lastly, open and close brackets. Inside the brackets, properties can be entered to define settings for a map.

Example: sky "<skytexture>" { properties }

Properties

Here are properties which can be assigned to each sky.

  • pic = "<graphic>"
<graphic> is the name of a background graphic or a cloud graphic to be used. It is unused for the fire sky which instead exclusively points to the "FIRE" graphic
  • backpic = "<graphic>"
<graphic> is the name of a graphic to overlay on the sky. This is used for mountains in Doom 64 and the graphic is placed 75 sky pixels above the bottom of the screen.
  • fogcolor = <hexnumber>
<hexnumber> is in the hexadecimal color format to define the color of fog. Most original Doom 64 skies have a "black" fog, however levels such as "Watch Your Step" and "In The Void" define a different color
  • fogfactor = <number>
<number> is a number to define how close the fog is to the player. Lower the number, the closer the fog is
  • basecolor = <hexnumber>
<hexnumber> is in the hexadecimal color format to define the background color of the sky. Sky graphics tend to cover this up, but it is particularly useful for the void sky type
  • highcolor = <hexnumber>
<hexnumber> is in the hexadecimal color format to define the color of the clouds or fire at the top of the screen
  • lowcolor = <hexnumber>
<hexnumber> is in the hexadecimal color format to define the color of the clouds or fire at the bottom of the effect
  • cloud
Enables the cloud sky effect
  • fire
Enables the fire sky effect
  • void
Enables the void sky
  • thunder
Enables the thunder effect
  • fadeinbackground
Sets the "pic" graphic to the bottom of the screen and allows a linedef action to fade it with the "EVIL" graphic.

Original Sky Definitions

Doom 64 used eleven different skies associated with the textures below.

  • F_SKYA - Violet colored clouds with thunder
  • F_SKYB - Red colored clouds
  • F_SKYC - Orange colored clouds with mountain backdrop
  • F_SKYD - Dynamic fire sky
  • F_SKYE - Orange colored clouds with heavy fog
  • F_SKYF - Space backdrop
  • F_SKYG - Space backdrop with Doom 64 logo (used for intro)
  • F_SKYH - Foggy teal void sky
  • F_SKYI - Dynamic green fire sky
  • F_SKYJ - Violet colored clouds with thunder and mountain backdrop
  • F_SKYK - Space and mountain backdrop

Examples

sky "F_SKYA"
{
    pic = "CLOUD"
    fogcolor = 0 0 0
    basecolor = b0 80 ff
    highcolor = 0 0 0
    lowcolor = 0 0 15
    cloud
    thunder
    fogfactor = 985
}
sky "F_SKYH"
{
    fogcolor = 0 40 40
    basecolor = 0 38 38
    highcolor = 0 0 0
    lowcolor = 0 0 0
    void
    fogfactor = 975
}

Sky Rendering Bugs

Doom 64 Remaster Color Bug

There is a bug in sky colors on the Doom 64 Remaster due to how it was translated over from EX to the KEX engine. Numbers like 1F are read as two different token types because of KEX's letter system (difference between how it reads numbers and letters). So fog colors like "8F 00 FF" will bug out because of the "8F". However, "89 00 FF" and "AF 00 FF" work just fine as they do not mix tokens. Token order matters too. If the hex value begins with a letter, it should be fine. It is only if it begins with a number and then a letter that causes a bug.

Doom 64 EX Sky Reference Bug

Doom 64 EX seems to be particular how skies are named. Sometimes it will render the texture for the sky and not the sky itself. Changing the name of a bugged sky in EX will often fix this issue.
